🏃 Run SFT (3 epochs)

Full fine-tune. Expected ~3-6 hours at 3 epochs.

Command to run inside container:

python models/code/train/sft.py \
  --model /workspace/models/merged \
  --train_file /workspace/models/Luminia-MiniCPM5-1B-Agent-GGUF/dataset/train_v4.jsonl \
  --out /workspace/train/outputs/sft_claude_agent \
  --epochs 3 \
  --neftune 5 \
  --bsz 1 \
  --accum 24 \
  --lr 1e-5 \
  --max_len 24576 \
  --train_cap 24576

Parameter Reference

--epochs 33 passes over 45,762 training rows --neftune 5Adds noise to embeddings (anti-overfit for multi-epoch) --bsz 1Microbatch size (1 per step, safe for VRAM) --accum 24Gradient accumulation steps (effective batch = 24) --lr 1e-5Learning rate (standard for full fine-tune) --max_len 24576Max sequence length (~24K tokens) --train_cap 24576Drop examples longer than this

Smoke test (5 minutes, verifies everything works):

python models/code/train/sft.py \

🧪 What to Check After SFT

  1. Loss curve — should start around ~2.5-3.0 and trend down below 1.0
  2. Generated output — test with test_prompts.txt using llama.cpp or a quick Python script
  3. Tool calling format — the model should emit <function> blocks for search/calculation prompts
  4. Reasoning retained — the model should still use <think> for complex questions

📋 After SFT: DPO

Once SFT finishes, the final stage is DPO (Direct Preference Optimization) to reinforce tool-calling behavior over stalling:

# 1. Generate on-policy preference pairs from SFT model
python models/code/data/build_prefs_onpolicy_gpu.py \
  --model /workspace/train/outputs/sft_claude_agent \
  --src /workspace/models/Luminia-MiniCPM5-1B-Agent-GGUF/dataset/train_v4.jsonl \
  --out /workspace/models/dataset/dpo_onpolicy_claude.jsonl

# 2. DPO training
python models/code/train/dpo.py \
  --model /workspace/train/outputs/sft_claude_agent \
  --data /workspace/models/dataset/dpo_onpolicy_claude.jsonl \
  --out /workspace/train/outputs/final-cognitive-core \
  --beta 0.1 \
  --lr 1e-6 \
  --epochs 3 \
  --accum 8

⏱ DPO takes ~2-4 hours. VRAM is tight (~22GB). If OOM, reduce --accum to 4.
